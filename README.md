# 🚀 OpenClash 全自动规则清洗与配置方案

本项目旨在通过 GitHub Actions 实现 OpenClash 规则的**自动化下载、深度清洗、分层去重及 YAML 配置动态生成**。通过这套方案，你可以彻底摆脱规则冲突和配置冗余的困扰。

---

## 🛠️ 主要功能
1.  **全自动构建**：每天凌晨 2:00 (北京时间) 自动拉取上游规则，支持随时手动触发编译。
2.  **智能去重逻辑**：
    * **优先级分层**：`手动直连 > 手动代理 > 指定策略组规则 (如 Google) > 通用直连 > 通用代理`。
    * 自动剔除低优先级列表中的重复项，确保规则执行逻辑清晰。
3.  **规则自动化清洗**：
    * 删除所有包含 `UDP` 的行（针对性优化）。
    * 自动去除注释 (`#`)、空行、行首尾多余空格。
4.  **YAML 动态生成**：根据你定义的规则源，自动生成策略组映射，无需手动编辑复杂的 YAML 代码。

## 📂 文件作用一览表

| 目录/文件 | 作用说明 | 修改建议 |
| :--- | :--- | :--- |
| **`rules-src/rules.list`** | **规则订阅清单**。管理所有上游下载链接。 | **核心修改项** |
| **`rules-src/*_custom.list`** | **个人自定义规则**。存放在此的规则优先级最高。 | **核心修改项** |
| `scripts/merge.sh` | 核心清洗逻辑。负责下载、去重、删 UDP。 | 除非改逻辑，否则不建议动 |
| `scripts/build_yaml.py` | 配置生成器。将规则映射到策略组。 | 除非改策略组名，否则不建议动 |
| `.github/workflows/` | GitHub 自动化流水线设置。 | 无需改动 |
| `rules/` | **最终产物**。清洗后的纯净版规则列表。 | **自动生成，请勿手动编辑** |
| `config/config.ini` | **最终产物**。OpenClash 直接调用的配置文件。 | **自动生成，请勿手动编辑** |
| `tmp/` | **Debug 目录**。记录了重复项和被剔除的列表。 | 仅用于排查问题 |

## 📝 如何配置与使用

### 第一步：修改规则订阅源
编辑 `rules-src/rules.list`，按以下格式添加链接：

* **`[Custom_link]`**: 需要**独立策略组**的源。
    * 格式：`名称|链接`（如 `Netflix|https://...`）。
* **`[Proxy-src_link]`**: 通用代理源链接。
* **`[Direct-src_link]`**: 通用直连源链接。

### 第二步：添加个人私藏规则
* 想要**必须代理**的域名或 IP：放入 `rules-src/Proxy_custom.list`。
* 想要**必须直连**的域名或 IP：放入 `rules-src/Direct_custom.list`。

### 第三步：手动触发编译 (可选)
如果你刚刚修改了源，不想等凌晨自动更新：
1.  进入 GitHub 仓库页面，点击顶部 **Actions** 菜单。
2.  点击左侧 **OpenClash Auto Build**。
3.  点击右侧 **Run workflow** 按钮。

## 🔍 如何排查规则问题 (Debug)

如果某个域名本该代理却直连了，你可以去 `tmp/` 文件夹查看这些“证据”：
* **`delete_proxy.list`**：存放了从通用代理中剔除的条目（因为它们已存在于直连列表）。
* **`Proxy_repeat.list`**：存放了被系统识别为重复的条目。
* **脚本日志**：在 Actions 的运行记录中可以看到每一个步骤的执行日志。

---

## ⚠️ 重要说明
1.  **私有仓库设置**：由于本仓库是 Private 的，OpenClash 访问 `rules/` 下的文件链接会报 404。**请务必在 OpenClash 插件设置中配置你的 GitHub Token。**
2.  **UDP 过滤**：为了防止某些服务（如游戏、测速）出现异常，脚本默认会删掉所有规则中带 `UDP` 的行。
